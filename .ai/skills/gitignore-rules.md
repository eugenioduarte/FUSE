# 🚫 .gitignore Rules — React Native (Expo)

> This document defines what must never be committed to the repository.
> Any addition to `.gitignore` must follow these categories and rationale.

---

# 🎯 Purpose

Prevent committing:

- Generated files that can be reproduced locally
- Native build artifacts (iOS/Android)
- Secrets and credentials
- Editor/OS noise
- Logs and temporary files

---

# 📦 Dependencies

```gitignore
node_modules/
.pnpm-store/
```

**Why:** Never commit installed packages. pnpm generates these automatically on `pnpm install`.

---

# 📱 Expo

```gitignore
.expo/
dist/
web-build/
expo-env.d.ts
```

**Why:**

- `.expo/` — local Expo dev server state, device registry, generated types
- `dist/` — Expo export output
- `web-build/` — Expo web build output
- `expo-env.d.ts` — auto-generated by Expo SDK, must not be manually edited or committed

---

# 🍎🤖 Native (iOS / Android)

```gitignore
ios/
android/
*.jks
*.p8
*.p12
*.key
*.mobileprovision
*.orig.*
```

**Why:**

- `ios/` and `android/` are generated by `expo prebuild` — never edit manually, never commit
- Certificate files (`.p12`, `.p8`, `.mobileprovision`, `.key`, `.jks`) are secrets — must never be in version control
- `*.orig.*` — generated by merge conflict resolution tools

---

# ⚡ Metro Bundler

```gitignore
.metro-health-check*
metro-cache/
```

**Why:** Metro cache is local machine state. Committing it causes conflicts and provides no value.

---

# 🏗 Build Artifacts

```gitignore
build/
*.ipa
*.apk
*.aab
```

**Why:** Binary build outputs. Generated by EAS Build or local builds. Too large and machine-specific for git.

---

# 🔷 TypeScript

```gitignore
*.tsbuildinfo
```

**Why:** TypeScript incremental build cache. Machine-specific and auto-regenerated.

---

# 🧪 Tests & Coverage

```gitignore
coverage/
jest-results/
test-results/
```

**Why:** Coverage reports are generated by `pnpm test`. Never commit — use CI artifacts instead.

---

# 🔐 Environment & Secrets

```gitignore
.env
.env.local
.env.*.local
.env.production
google-services.json
GoogleService-Info.plist
```

**Why:** These files contain API keys, Firebase credentials, and environment-specific config.
Committing them is a critical security violation.

**Rule:** All secrets must be injected via EAS Secrets or a secure vault — never hardcoded or committed.

---

# 🚀 EAS (Expo Application Services)

```gitignore
.easignore
eas-build-pre-install.sh
```

**Why:** EAS-specific build scripts may contain environment references or be machine-generated.

---

# 💻 Editor & OS

```gitignore
.DS_Store
Thumbs.db
.vscode/
.idea/
*.swp
*.swo
*~
```

**Why:** Editor and OS metadata is personal and irrelevant to the codebase.
Note: `.vscode/` is ignored globally — if the team shares VSCode settings, use `.vscode/settings.json` whitelisted explicitly.

---

# 📋 Logs

```gitignore
*.log
npm-debug.log*
yarn-debug.log*
yarn-error.log*
pnpm-debug.log*
.pnpm-debug.log*
logs/
```

**Why:** Log files are ephemeral debugging artifacts. Never useful in version control.

---

# 🛠 Misc Tooling

```gitignore
.turbo/
.cache/
tmp/
.tmp/
```

**Why:** Local cache directories from Turborepo, build tools, and temp files.

---

# 📁 Legacy / Excluded Projects

```gitignore
Fuse/
```

**Why:** `Fuse/` is a legacy project at the repo root kept for reference only — must not be tracked.

---

# 🔒 Absolute Rules

1. **Never commit** `google-services.json` or `GoogleService-Info.plist`
2. **Never commit** any `.env` file regardless of environment
3. **Never commit** `ios/` or `android/` — always generated via `expo prebuild`
4. **Never commit** certificate files (`.p12`, `.p8`, `.mobileprovision`, `.key`, `.jks`)
5. When adding a new tool that generates local artifacts, add its output to `.gitignore` immediately

---

# 🧭 How to Update `.gitignore`

When a new tool, dependency, or workflow introduces local-only artifacts:

1. Identify the generated file/folder
2. Add it to the correct category section in `.gitignore`
3. Add the corresponding entry to this skill with **why** it must be ignored
4. Commit with: `chore(config): update gitignore for <tool-name>`
